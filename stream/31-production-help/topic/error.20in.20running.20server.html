Robert Hönig (Jan 11 2017 at 22:22):

@Tim Abbott The issue is probably to be found in two different ways of installing pip:

  • via pip install pip
  • via apt-get pip

Those two methods stored pip at different locations:

  • /usr/local/bin/pip
  • /usr/bin/pip

So all of our efforts to update pip actually did update pip, but /usr/local/bin/pip, where as the system searched pip in /urs/bin/pip (which pip).

The issue got resolved by running hash -r, which cleaned the cache of the bash and thus the wrong path that prevailed after updating.
