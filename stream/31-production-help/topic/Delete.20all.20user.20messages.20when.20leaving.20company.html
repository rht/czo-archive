Mateusz Mandera (Jun 16 2022 at 14:05):

@Unam Hmm, might need to script something - there is the delete_user management command that cleans up user messages, but it does it while deleting the actual user - while it sounds like you want to re-use accounts :thinking: Is there a reason you need to actually re-use the generic accounts though? Wouldn't it be simpler (and perhaps safer?) to create a new generic account for each intern? (And old accounts can be deleted via delete_user as mentioned)

Alex Vandiver (Jun 16 2022 at 17:36):

This is precisely a reason why account reuse is generally not suggested -- there are a large number of nooks and crannies you have to figure out how to clean out, otherwise you're potentially sharing all kinds of data.

I'd strongly counsel you to make each intern their own account.
