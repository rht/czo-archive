Richard (May 28 2020 at 16:37):

It looks like your database user is running in LATIN1 encoding, which I believe isn't something Django is designed for.
While this isn't Zulip (or even Ubuntu) specific, it does give the way to change client encoding to the UTF8 listed in the error: https://www.digitalocean.com/community/tutorials/how-to-use-postgresql-with-your-django-application-on-centos-7

I suspect you'll need the command that sets client_encoding, but modified for your zulip database user:
ALTER ROLE myprojectuser SET client_encoding TO 'utf8';
