Shayan Toqraee (Apr 22 2017 at 18:08):

@Tim Abbott
Sorry for my delay.
Fiddler is a proxy server. It by default works locally, and sets the default proxy configuration of Windows to go through it. And it shows all the traffic that is going through. They call it a network debugger. (It also works on Linux by using Mono.)
One very interesting thing that Fiddler does is that it can decrypt the HTTPS traffic by being a "man in the middle". It gives you its root certificate to add it to the trusted root certificates of the browser or OS, so the connections do not complain that the connection is not secure.
What I did was that I changed Fiddler's settings so that it allows remote connections, and added a proxy setting to the WiFi connection settings of the android device. I also added Fiddler's root certificate to the device the same way I added my own root certificate. Fiddler was able to get all the traffic and decrypt it. And everything worked fine this way.
Is there any differences in the security of the connection when I connected directly or through Fiddler? Both need a root certificate to be installed on the device.
(The certificate that my Zulip server uses is not itself loaded on the device, the root certificate that was used to sign it was loaded. Fiddler also works the same way.)
