Topic: User Email Address Change with Okta SSO

Jeremiah (Jan 25 2023 at 19:29):

We are running Zulip Server version 6. The only Authentication method we have enabled is with Okta SSO. One of my users primary email address has changed and the next time they tried to signin to Zulip it saying that his account has been deactivated.
We have tried to change the Zulip email address in Okta, but that didnt fix it.

While it would be nice to keep this users past message history, the more important thing would be to get them back on the system. Any Ideas on how to fix this?

Jeremiah (Jan 25 2023 at 20:11):

We are not opposed to binging the accounts local and not using Okta SSO if needed. It is a small team and we can make it work if that is possible.

Tim Abbott (Jan 25 2023 at 20:51):

@Jeremiah you can use the change_user_email managemnet command documented here to align the user's email address in Zulip with that in Okta: https://zulip.readthedocs.io/en/latest/production/management-commands.html

Jeremiah (Jan 25 2023 at 22:40):

@Tim Abbott That worked. Thank you very much. For some reason the new email address was in the database already and I
was able to change it, and then change this users email. The user was able to login fine.
