Some problems with this solution:
1: System fonts don't have glyphs for all the emojis so some emojis will get rendered as image, some as native emojis and some as empty boxes. We faced same problem in twitter integration.
2: It is not possible to change the image based on the emojiset of the user.
3: As already pointed out, realm emoji will not get rendered.

I had also tried to make this happen in webapp notifications back in November but didn't find any satisfactory solution.
