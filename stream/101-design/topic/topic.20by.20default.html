Rishi Gupta (Nov 28 2017 at 21:41):

One big issue new folks have with zulip is confusion with interleaved views. What would folks think of the following change:

  • Clicking on a stream name shows only the first topic.
  • You have to click a second time (or use the s shortcut) to see the full stream view

I think the main cost for experienced users is that when catching up on streams you don't care about, often you do want the stream-level interleaved view, so that you can hit End if nothing else. Also the double-click thing will be pretty hard to discover.

(Note: If we do do such a change, I think we would absolutely want to try it on chat.zulip.org for a few days first :).)

Steve Howell (Nov 28 2017 at 22:07):

So I'm definitely biased here by the way my brain works, but I've never been a big fan of the interleaved view. So I support a more topic-centric approach in Zulip. I'm 100% in favor of having clicking on a stream name show the first topic, which I assume you mean is the first one in the sidebar, aka the most recent topic. In order to widen, I think we want to think about making a more obvious mechanism than the s shortcut to do it. You can click on the recipient bar, of course, but that's not totally obvious either.

Steve Howell (Nov 28 2017 at 22:08):
