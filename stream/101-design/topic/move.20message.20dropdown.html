Neil Pilgrim (neiljp) (Aug 17 2022 at 23:12):

I was just looking at implementing this for ZT, which will be straightforward and I'll do shortly.

It's after the change, but for me, things that are definitely better with these strings:

  • Same verb at start of each option (depending on Ui, could be a title/caption)
  • Clarification on which messages are affected ('in this topic' vs previously 'to this topic')
  • Simplification ('all messages' vs previously 'also'+'previous and following')

However, I'm stumbling on 'move', since I think it's used inconsistently:

  • A quick scan indicates the help center says we can 'move' some messages to a different topic, or 'move' an entire topic to another stream, but 'rename' an entire topic (only within a stream?)
  • The UI has 'move'-topic in a popup from a topic 3-dot menu, which allows full-topic between- and in-stream moving/renaming.
  • The UI has an 'edit' action for entering the part discussed above, which is both single-message content editing, as well as the 'move' options for the encompassing topic (one/one+later moving and moving all - renaming in the help)

So, I think good follow-ups here would be to:

  • distinguish move+rename: decide whether to use both rename and move as now or differently, in the help/UI; perhaps add 'rename' to the UI, or maybe in the help use rename as a document title for easier lookup but explicitly describe it as a move. Maybe the help would benefit from a 'Reorganizing messages and topics' summary/section for this?
  • distinguish edit content from move/rename topic: shift this dropdown closer to stream/topic and away from content, or otherwise show their association/separation
  • ultimately determine if there's a simpler UI which would allow separation of single-message edit-content from the 1+-message topic edit (I'm guessing this been discussed already?)
