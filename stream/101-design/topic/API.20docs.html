Tim Abbott (Nov 08 2017 at 21:22):

Hey folks, exciting news! We now have a version of the replacement for /api that should be possible for several people to collaborate on. It's based on the /help/ site infrastructure and is available at /api-new/: https://chat.zulip.org/api-new/

Obviously a lot more work is needed to make this good:

  • Adding a lot more endpoints to the "usage" section
  • Adding links to the repositories for the binding projects.
  • Maybe adding Swagger links
  • Moving the guides for writing bots and integrations to the new /api-new site (right now most of the links take you to ReadTheDocs)
  • Moving it to take over the /api URL.
  • Etc.

The hope is that once we swap this in for the old /api, it'll be possible for several folks to collaboratively iterate on this. I'd absolutely love it if a few folks played with the docs and tried to find anything that's worse in the new version than the old /api; hopefully we can fix those rapidly and then swap it in.
