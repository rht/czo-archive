Neil Pilgrim (neiljp) (Jan 03 2022 at 21:17):

For comparison, in regular color schemes, ZT currently has:

  • a fully colored box for selection (background colored)
  • darker message text for unreads
  • regular message styling for read messages

I'm not suggesting a full colored box for selecting generally; in ZT this enables us to emphasize the selection without row/column changes in the terminal.

However, one nuance about the 'darker for unreads' approach, which is currently taken to a strange extreme in our monochrome theme (due to very limited styles being available!), is to make unread messages "intentionally unreadable" (or at least less readable) - in monochrome they have a strikethrough style applied right now. Currently ZT has a more "intentional reading" approach to messages; whether that ever happens in the web app, having eg. a message background fade from some other color to the "normal" color (improving contrast in doing so) would probably also help make the reading process more obvious. Of course it needs to be obvious that there are unread messages sitting there.

Greg Price said:

In my usage I find the unread-message indicator helps quite a lot in quickly orienting me when I'm looking at an old thread that has some new messages -- which is a large fraction of the threads I read. One thing about it is that because it's just at one spot in the message, rather than being continuous along the message's height, it's not as visually dependable. So I have to take extra time to carefully scan upward to find there's definitely a message-top up there and there's still no dot there.

Certainly this is something we could try an experiment to learn more about, though.

Tim Abbott (Jan 13 2022 at 23:56):

In early Zulip development we played with a bunch of visual designs for this cursor, including an arrow indicator pointing to the left side of the message feed, highlighting the message in a different color, and a few others.

Tim Abbott (Jan 13 2022 at 23:58):

We ended up with an outline/box after testing a few options for these reasons:

  • Highlighting the entire message is a lot more visually distracting, and limits one's ability to happily use color in message bodies for other purposes (E.g. images in the message will look off with a blue highlight over the entire message, etc.).
  • A cursor to the side of the message can be pretty invisible, and also doesn't clearly communicate the top/bottom boundaries of the message, which is important for predicting what will happen when you use keyboard shortcuts that interact with a message. (E.g. if you're trying to quote a particular paragraph, you want to be certain whether that paragraph is inside the message you have selected or not).

Tim Abbott (Jan 13 2022 at 23:59):

I think those arguments generally point to something that has vertical height equal to the height of the message. But I think we can play with a bunch of options subject to those constraints -- e.g. one could try having left/right borders only, or left/right borders that much bolder than top/bottom borders, or dashed lines -- and I think we should make screenshots with a few variants and see if any seem promising.

Tim Abbott (Jan 14 2022 at 00:01):

There's a separate question of whether we want to have a "unlikely to use keyboard shortcuts" mode that doesn't display it at all, for example, but I think it makes sense to design the best version of this visual indicator for the many users who do use keyboard shortcuts independent of that question.
