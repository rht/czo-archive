Originally the list was going to be pretty short in order to prevent false positives, but it would be great to hear suggestions as to how long the list of translate-able emoticons should be as well as which emoticons Zulip users want to be translated. Rishi Gupta (Jan 21 2018 at 18:09):

to be concrete, here are some lists of emoticons:

  • https://get.slack.help/hc/en-us/articles/202931348-Emoji-and-emoticons#use-emoticons
  • https://aprescott.com/posts/hangouts-emoji
  • https://en.wikipedia.org/wiki/List_of_emoticons.

My first thought is to support a fair number of emoji, but not too many alternates for each one. Some potential cons of too many, besides false positives:

  • markdown performance (not sure how big a deal this is)
  • if the main use case is that people have some particular few emojis they use a lot, like :smile:, and they just want a fast way to type it, then having a lot of alternative emoticons for it doesn't add much value
  • a smaller list makes reading the documentation for it less overwhelming
