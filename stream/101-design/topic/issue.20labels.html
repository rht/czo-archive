Alya Abbott (Apr 25 2022 at 17:01):

Good question!

  1. Any redesign issue that's important and/or a blocker for other work should get the "priority: high" label. I think that will be the case for most issues you file.
  2. "Help wanted" means the issue is ready to be worked on by a contributor. So you might assign it in many cases, but not, e.g., if the issue needs some feedback from me, or is difficult and likely needs to be taken on by a core team member.
  3. "Release goal" means that something is specifically a goal for the 6.0 release (see #general > Zulip 6.0 release plan ). Tim Abbott and I will add that label if appropriate; many redesign issues will be high-priority but not specifically release goals.
