Tim Abbott (Mar 15 2023 at 22:55):

I think it'd be quite valuable to preserve the fact that a one-line message break is the same height as a paragraph break, and I don't see a technical reason why the changes we have deployed here should need to change that -- we've always had a blue box and previously were able to make this work, and actually the current blue box is thinner than the original.

It seems like an implementation bug. Further, if you look closely, the blue box isn't centered properly around 1-line messages in this currently deployed version (there's more space above the text than below, which can be visually distracting) -- so that's probably a (related) bug.

To talk about the philosophy here: A lot of the time, one is posting a series of thoughts on something, each in their own paragraph, and if nobody else is posting messages, we want it to feel like your decisions about when to hit enter and continue in a next message can be based on editorial considerations -- things like whether this message is already a complete thought that you'd like folks to be able to emoji react to and your next paragraph is going to be an independent thought, or if you want to let others see what you've written so far as you write the rest to provide a better real-time experience for them, etc., without changing materially the visual experience of reading your series of paragraphs.

Having slightly different spaces also just looks visually less nice -- it can be distracting to see a series of paragraphs that have mixed/choppy spacing. (Every time we regress this, we start getting a slow stream of bug reports "these paragraph breaks look funny", and I think there's a reason for that ... and I even remember some folks wanting to do things like edit messages to copy/paste the later ones into the first one and then delete the rest to get the paragraph spacing they want, which seems like a bad outcome).
