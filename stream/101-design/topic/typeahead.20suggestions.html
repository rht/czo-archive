Stream: design

Topic: typeahead suggestions

Rishi Gupta (Feb 24 2019 at 08:11):

Currently we only show typeahead suggestions after the user has entered a character. E.g. after @x rather than @. I was a proponent for the character after for a while, but after watching someone use Zulip today I realized that if you're not confident you're doing the right thing, having the typeahead show up immediately is nice. I think it's also compelling that other products seem to do it this way (tested github, slack, paper).

Thoughts?

Armaan Ahluwalia (Feb 24 2019 at 08:41):

I agree. It happens to me often with at and : across different products I like to confirm it’s possible by seeing a popup

Steve Howell (Feb 24 2019 at 12:30):

I don't have a strong opinion here, but as long as there is some logical way to make suggestions on zero characters, which there almost always is, I say we should do it.

Armaan Ahluwalia (Feb 25 2019 at 02:11):

Usually alphabetical should be pretty logical or better yet contextual. But alphabetical is a good start.

Tim Abbott (Feb 25 2019 at 19:37):

I feel like having early-mention is a behavior which can be alternately annoying or useful depending on what you're trying to do and how good the suggestions are. We might need to implement a different algorithm for that 0-character case than we want with a single character (because it should probably do alphabetical last in order to be net useful).
