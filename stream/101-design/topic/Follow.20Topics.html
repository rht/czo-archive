Currently:

  • Open two tabs. In the first one, open /#settings/topics. From the other tab, mute/unmute a topic. This change is not reflected in the settings panel in the other tab.

I have fixed this issue in #26016

Still, I would like to point out the tradeoff we made:

In #25127, we removed this code block.(which resulted in the bug)

The expected behaviour was:

2. Let's not have the topics get resorted when you change the status in this UI, as it makes it harder to
find what you were looking at. The order should just get updated when you navigate away or refresh.

3. When you set a topic to "Default for stream", it should stay in the UI until you refresh or navigate away.
This makes it much easier to review your changes, and undo if needed. It's the same pattern we follow in
the Users panel when an admin deactivates a user, for example.

The tradeoff is:

  • Even if we change the status from the topic popover or message header bar only the status will be updated (It won't get re-sorted by date updated or removed on changing the status to "default for stream").
