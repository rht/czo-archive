Vlad Korobov (Jul 31 2022 at 01:17):

@Aman Agrawal (amanagr) this would be relevant to what you do. @Greg Price @Anders Kaseorg you might be opinionated about that area, so check this out.

In the new design if Figma and here https://github.com/zulip/zulip/issues/22021 the way recipient bar colors should be generated isn't clarified. And Aman's PR helped me to realize that.

I made a prototype to solve this problem https://codepen.io/terpimost/pen/WNzdrJN?editors=0010 and also recorded video clarifications
Basically I suggest to clamp highness of a stream color, at least on a front-end side:

function correctStreamColor({ color = "#000000"}) {
  let color_l, min_color_l, max_color_l;
  min_color_l = 20;
  max_color_l = 75;
  color_l = chroma(color).get("lch.l");
  if (color_l < min_color_l) {
    return chroma(color).set("lch.l", min_color_l).hex();
  } else if (color_l > max_color_l) {
    return chroma(color).set("lch.l", max_color_l).hex();
  } else {
    return color;
  }
}

and for the recipient bar to mix it without relying on browser blending and transparency:

function getRecipientBarColor({ color = "#000000", darkMode = false }) {
  return chroma.mix(color, darkMode ? "black" : "white", 0.8, "rgb").hex();
}

I was using 0-dependency library https://vis4.net/chromajs/ so simplify color manipulations
If you want to clamp stream colors on the server side - ok. It would be even possible to use 2 colors as we do now and avoid any front-ent js scripting about it. I expect some color manipulation for counters and probably will create a prototype for them tomorrow.

Anders Kaseorg (Mar 03 2023 at 20:32):

Surely you want LAB, the Cartesian counterpart of the polar LCh space. LCh has singularities at black and white:

> chroma("#000000").lch()
[ 0, 0, NaN ]
> chroma("#ffffff").lch()
[ 100.00000386666655, 0.00001795054880958058, NaN ]

so mixing in LCh is highly discontinuous in those regions:

> chroma.mix("#00ff00", "#000000", 0.7, "lch").hex()
'#005200'
> chroma.mix("#00ff00", "#010000", 0.7, "lch").hex()
'#643011'
> chroma.mix("#00ff00", "#000001", 0.7, "lch").hex()
'#004770'

vs. LAB:

> chroma.mix("#00ff00", "#000000", 0.7, "lab").hex()
'#1d4715'
> chroma.mix("#00ff00", "#010000", 0.7, "lab").hex()
'#1e4715'
> chroma.mix("#00ff00", "#000001", 0.7, "lab").hex()
'#1d4716'

Anders Kaseorg (Mar 03 2023 at 20:34):

LCh is useful for producing things like rainbow gradients, but you generally want LAB for perceptual mixing, which is why colord’s mix plugin works that way.
