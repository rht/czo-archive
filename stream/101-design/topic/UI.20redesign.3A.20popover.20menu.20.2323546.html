Tim Abbott (Nov 17 2022 at 19:00):

Just to talk a bit about why this is important: Renaming a feature is a complex project in itself, since it requires coordinated changes across the UI, documentation, etc., and generally we want to do that sort of thing as a single independent project, so we can check our work on it and make sure we made the change correctly and completely.

One of the most important parts of creating work that can be merged into a large project like Zulip is figuring out a scope for a change such that one can integrate chunk having verified that each one has no regressions.
