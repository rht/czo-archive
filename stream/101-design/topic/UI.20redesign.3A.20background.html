Alya Abbott (Jan 11 2022 at 19:32):

Hey folks! I'm splitting out this thread to specifically discuss the background color changes proposed in #design > UI redesign .

In particular, Vlad Korobov 's most recent prototype makes the following changes in light mode:

  1. Changes the overall background color to a light-ish grey, with the message feed cards keeping their white background. This creates more visual separation between the sidebars and the message feed, and also between sections in the message feed.
  2. Changes the color of a selected section in the left sidebar (e.g. stream or PMs) to white. There's bunch of specifics regarding selections that we should discuss, which I'll call out in a follow-up message.
  3. Makes the background for unread counts lighter than at present (but still a bit darker than the background grey color), and changes the font from white to black.
  4. Changes the background for the compose area to a light grey (lighter than the sidebars, matching the proposed nav bar color), both when collapsed and when expanded. (We can think about whether this change can be made later.)

I assume we would want to make a parallel set of changes in dark mode, but I think it's fine to keep the discussion focused on light mode for now.

Andy Stewart (Jan 11 2022 at 23:06):

I love the change to a light gray neutral background color, and a white background for the cards.
I think we reconsider the :hover and :active colors to a more neutral setup as well.

So a white opacity or dark opacity change rather than including the blue. Reasoning:
• ensure there is a distinct value change (light blue hover next to light gray background can carry too similar of a color value contrast)
• does not clash with or contradict stream color and stream color hashtag graphic
• allows other color states to have increased hierarchy (links, marker, streams)
• perhaps simplifies the layout further by reducing color noise
