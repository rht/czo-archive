Topic: Return cursor back to its original position (link click)

Garvit Dewan (Nov 28 2017 at 07:03):

So this is in reference to #6971.

The idea is to return back the cursor (in case it is in a typing position in any input field) to its original position when a link is clicked on the page. Currently, it returns back only to the compose, wherever you might previously be.

So, we'd like to have some suggestions on whether to keep it all throughout the UI of the chat, or only when the compose box is open (follow #6971 to get a gist of the discussion in detail). we probably want this in the 3 search UIs on the main page, but we'd like some user feedback on it before proceeding.

Tim Abbott (Nov 28 2017 at 22:32):

@Greg Price this is specifically about when you have compose open and click a link in the message feed. By default, in a browser, clicking a link like that moves the browser focus (cursor) to that link, so you'd lose your place in the compose box if you wanted to keep typing.
