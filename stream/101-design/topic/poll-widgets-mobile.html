The heading looks actually bigger than on the webapp. We can probably dial it back to the same size.

True, I was having doubts about its size.

The border, and associated padding, make the available space narrower. We already have less space to work with than the webapp, so it's likely best to leave that out.

I think the border is important to keep, since they do make the messages 'widgets', and as you said, differentiates between a normal message, and the other.

Since you said later that the 'votes button' needs to go (which I agree with completely), these borders might provide a nice framework to put all the options in -- without them, we'd have a lesser number of ways we could properly display the options. The design I have in mind, is this -
- The box which currently exists, stays. The insides get changed though.
- The heading remains the same -- with the font-size change
- The options appear as a list, without any apparent button on screen -- so only the text, listed.
- Once you click an option, it would have the ripple effect, and the option would essentially be voted on.
- Any option which the user has voted for will have it's row in the list highlighted (the same way the reactions get highlighted currently in the mobile app).
- The number of votes would be on the right of each row, with a lower opacity.
- The user would be able to see who voted for each option by long pressing on it.

I will post a mockup of this design soon in case it's hard to understand.

The border under the poll question feels like too much separation. Remember that it's a question and the options are answers... so they're basically consecutive sentences in a conversation.

That is true. Can we perhaps lower it's opacity, and it'll improve? Like this?
