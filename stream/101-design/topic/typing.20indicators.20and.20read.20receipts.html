Tim Abbott (Jun 09 2021 at 00:31):

FWIW, I think "read receipts" with data synchronization and a WhatsApp-style UI is a lot of work, and likely not an investment that make sense at this time. The real-time synchronization piece is similar to typing notifications and not too complicated, but we also need to be able to efficiently return the set of users who have read messages in our API requests to fetch 1000 messages of history, and that will look like an additional copy of the Emoji Reactions feature (the issue has notes on a design for that piece).

If we want to have a privacy setting and the simpler "You can get the information with a click if you need" approach, which at least some users on the issue thought would solve their problem, that's a pretty small project, probably 10x less work in Zulip than a WhatsApp-style UI where the "read receipts" status is immediately visible.
