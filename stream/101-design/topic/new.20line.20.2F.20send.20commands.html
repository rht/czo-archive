Screen-Shot-2021-12-02-at-1.12.19-PM.png

The shortcuts in the ? menu are different:

Screen-Shot-2021-12-02-at-1.08.27-PM.png

In terms of what actually works (on my Mac):
1) Cmd + Enter, Ctrl + Enter, and Shift + Enter all insert a new line if "Press Enter to send" is enabled.
2) Cmd + Enter and Ctrl + Enter send the message if "Press Enter to send" is disabled. Shift + Enter still inserts a new line.

As I'm writing this, I realize that the nice thing about suggesting Shift-Enter for inserting a line break in ? is that it always works. On the other hand, in the switcher widget, it's good to have the text remain the same when you switch. So perhaps it's fine to suggest Shift for inserting line breaks in ?, while suggesting Ctrl in the switcher.

But we probably need to change the switcher widget text for Mac users.

1) I think we should be consistent about referring to Cmd, and not Ctrl, for Mac users. See also #20107.
2) I got a bit lost when trying to understand Enter vs. Return. Should we replace Enter with Return (like it says on the Mac keyboard) for Mac users in the new switcher, the way we do in the ? menu?

1) The line below the Send button says either "Enter to send" or "Ctrl + Enter to send".
2) There is a \vdots menu to the right of that line.
3) The menu has two options, with radio buttons to select between them:
o Ctrl + Enter to send
Enter to add a new line
o Enter to send
Shift + Enter to add a new line

Bold might not be the right formatting, but the idea is that the first line of each option is more salient compared to the second line.

  • Avoiding having too much text/clutter in the base compose box.
  • Making it really clear what the current state/instructions are.

  • Having a thing that's clearly meant to be clicked on for how to change the configuration.
