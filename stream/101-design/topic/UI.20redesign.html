Anders Kaseorg (Dec 23 2021 at 10:23):

This is awesome Vlad! Very good overall direction—much cleaner and more modern while still being Zulip.

To address some of the questions you brought up in the video:

  • Our main scrollbar is on the browser’s right so that users with large screens have the entire browser window as a target area where they can use the scroll wheel or touchpad scroll gestures. Visually moving the scrollbar without breaking this would require some kind of kludge that listens for scroll events and manually translates them in code to scrolling movement; such emulation inevitably results in laggy awkward behavior that feels much worse than the platform’s native scrolling.
  • The Zulip logo in the top-left corner is customizable, and we expect organizations to replace it with their own logo, but it would be reasonable to streamline the default logo we use there.
  • The multiple new-message buttons are an effort to nudge new users toward keeping the discussion organized. There’s a concern, for example, that if starting a new topic were to take more clicks or be less discoverable than replying to an existing topic, it would become much more common for new users to randomly append new discussions to unrelated topics. There’s room for different ideas here; we’d need to do user testing to be sure.
