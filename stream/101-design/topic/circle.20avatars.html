Anders Kaseorg (Feb 23 2022 at 00:48):

I made this one in Python:

import numpy as np
from PIL import Image

def generate_avatar(seed: int, size: int) -> Image:
    rng = np.random.RandomState(seed)
    bg = 255 - np.diff(np.sort(rng.randint(0, 256, 3)), prepend=0)
    fg = np.diff(np.sort(rng.randint(0, 256, 3)), prepend=0)
    i, j = np.ogrid[:9, :5]
    noise = rng.rand(9, 5) - np.hypot(i - 4, j - 4) / 4.5
    symmetric = np.c_[noise, noise[:, -2::-1]]
    field = np.asarray(Image.fromarray(symmetric).resize((size + 1, size + 1), Image.BICUBIC))
    a, b, c, d = field[:-1, :-1], field[:-1, 1:], field[1:, :-1], field[1:, 1:]
    weight = np.clip(((a + b + c + d) / np.hypot(a - b + c - d, a + b - c - d) + 1) / 2, 0, 1)
    color = bg[None, None] + (fg - bg)[None, None] * weight[:, :, None]
    return Image.fromarray(np.rint(color).astype(np.uint8))

Tim Abbott (Feb 23 2022 at 20:50):

Alya Abbott said:

Would there still be a way to upload a picture once and have it used across all Zulip orgs, the way you can with gravatars?

This is kinda a hard problem if you want to prioritize privacy. With Gravatar, the model is that it contains a hash of your email address, and thus fundamentally if you can see someone's Gravatar, you can brute force your email address. The simplest models for doing this sort of thing would be to make the avatar be a function of something like a (server_uri, user_id) pair, or perhaps a realm_uuid, user_id pair (if we want it to be stable across domain changes), and that of course won't follow you to other hosting.

Gravatar's origin is as a way to have the same avatar across a bunch of Wordpress blogs all over the Internet, and was created with that design constraint.

Tim Abbott (Feb 23 2022 at 20:50):

If we want our procedurally avatars to be stable across different Zulip Cloud orgs, we'd need to make it a function of something stable in that way, which basically has to be your email address, which we don't want to be sharing with all users in every Zulip Cloud organization. But I think that'd be basically a pointless feature: What you want is an avatar you uploaded to be used everywhere, not a procedurally generated one.

Tim Abbott (Feb 23 2022 at 20:57):

Right, I should also add that for Zulip Cloud some specific details that probably not everyone here knows:

  • When you create a second account with the same email address, you're offered the opportunity to import your avatar from any existing Zulip Cloud organization. This has been true for 2-3 years. (It also will import most other personal preference settings). The thing that isn't great today is that if you decide to change your avatar, you have to upload it in each organization, which is annoying, but for a relatively infrequent task. This is a consequence of our decision to have accounts be per-organization, which is good for the security of corporate users (E.g. you can require SAML authentication to login to a specific organization) but results in this sort of inconvenience for non-corporate organizations.
  • There's no simple way to share Zulip Cloud avatars with self-hosted organizations without some sort of security/privacy consequences. But it would be possible in a future OAuth world.
