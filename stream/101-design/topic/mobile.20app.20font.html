Isham Mahajan (Jul 03 2019 at 13:12):

There has been a very detailed discussion already about whether we should include a font within the mobile app in various mediums (in #M2447, and in CZO at #mobile > input height discrepancy -- please go through those if possible). I wanted to shift the discussion to here to get the opinions of the entire community and partially also to revive it. There is another point I would like to add before starting this discussion -
- As I mentioned in #M3533,

I think [it] is a very good thing to do because of other reasons as well -- primarily because every messaging app now-a-days has a unique font, which adds a certain unique flavor to it, although what font to choose might be a huge debate in itself.

For example, Discord and Slack.

Here's an overview of the other reasons which can be found in the provided links as well.
- There are several bugs which exist with react native itself, where custom fonts provided by the device doesn't scale properly to fit the boxes the app provides for the text to be placed.
-

Fonts differ in how much vertical padding they build in -- for the sake of ascenders and descenders in Latin letters, or more so for some other scripts like देवनागरी that use more vertical space -- and that can cause the same text widget, with all the same properties right down to the font size, to lay out with a different height when given a different font. That in turn makes it quite difficult to design a layout that consistently looks right.

-

An extreme example is #2365 -- a component we get from one of our dependencies comes out completely broken with the default font on certain devices.
