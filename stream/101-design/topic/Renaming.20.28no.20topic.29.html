My perspective is there's no reason to have the word "topic" in the "general chat" topic, and there are downsides to it being there. I guess my main concern is that we want this to serve as an example of a good topic name, ideally, and good topics generally don't have the word "topic" placed at the end of them.

Tim Abbott (Jan 06 2023 at 02:20):

We should also think about considerations around users wanting to refer to this feature. For example, "Please don't send cat pictures to the general chat" is a thing someone might want to feel normal saying. "Please don't send cat pictures to the empty topic" might feel a bit weird.

The status quo of "Please don't send cat pictures to (no topic)" is pretty bad on that front :smile:. That choice was motivated by the idea that we want to push people to actually use topics, and so there's some benefit to having one feel like you're it wrong if you send things there.

Tim Abbott (Jan 06 2023 at 02:20):

Tim Abbott (Jan 06 2023 at 02:21):

Tim Abbott (Jan 06 2023 at 02:25):

I think my preferred option is "general chat". I don't think it's perfect -- what do you send to the "general chat"? But maybe that's fine given the concept is that if you want to talk about something in particular, you should probably put it in a topic... But if you just want to chat, that's the place to chat.

It seems like this might involve a lot of steps, which haven't been discussed in depth, so I'm going to try to write out at least some of them here:

  • Switch from storing "(no topic)" in the backend to storing an empty string, and migrate existing messages to empty string. Have the frontend treat empty string as a place to display "(no topic)", translated
  • Change the string that we display in the frontend from "(no topic)" to "general chat" (also a translated string)
  • Probably also ban any messages from being stored on the backend with topics "(no topic)" or "general chat" since I think that would get confusing. We could just automatically convert those to empty string
  • Prevent empty string topics from being resolved
  • Still allow messages to be moved out of the general topic
  • Probably still allow messages to be moved into the general topic? But I like the idea of adding a layer where, if the box is empty, confirm with the user that they want to send the message to the general topic.

Thoughts?
