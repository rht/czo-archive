Alya Abbott (Feb 02 2022 at 17:32):

For reference, this is the original issue: #18728.

The n ("Next unread topic") and p ("Next unread private message") keyboard shortcuts are a great way to read through one's Zulip messages. At present, we don't have buttons that correspond to these shortcuts. Adding these buttons would make this workflow more accessible to mouse users, and more discoverable.

We should therefore add the following buttons:

  1. "Next unread topic (n)" button in the blank space at the bottom of a topic narrow. No button if there are no unread topics.
  2. "Next unread PM thread (p) button in the blank space at the bottom of a PM conversation narrow. No button if there are no unread PM threads.

We will need to experiment with the design and exact placement of these buttons.

Tim Abbott (Feb 08 2022 at 02:24):

A few notes on details we will need to think about/resolve:

  • We have other things we put at the end of the message feed, such as a loading indicator in rare cases if you started in a search high up and went to the bottom, and a "Subscribe" button if you're looking at a stream you're not subscribed to. How should this relate to those? My initial answer is this should always be after these things (including bit of vertical buffer we leave for the loading indicator so that it can disappear without anything changing position).
  • How do we handle the "Next unread topic" vs. "Next unread PM" issue -- e.g. should what we offer depend on whether you're currently looking at a topic or a PM conversation? (Or do we show both, which would make the element a lot bigger?)
  • How do we handle the interactions if you have no unreads of the relevant type? Does it just result in a thing you can click that doesn't do anything, or that pops a small "You've read everything" tooltip, or something else?
  • There's a potential interaction with this specific position that it will be an element that moves down every time a new message arrives, and that could be annoying/distracting. I don't think we can easily evaluate whether that'll be a real problem until we try it.
