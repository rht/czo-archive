Greg Price (Jul 01 2021 at 23:27):

I think one habit I have in using Zulip (and @Alexx Henry, you might or might not find this helpful) is that before typing a navigation command like c or n or t, I'll hit the Up or Down arrow key a couple of times.

This has the effect of closing the compose box, if it's open but empty. And if it's already closed, it's basically harmless, just moves the blue-box cursor a couple of messages.

For me, I think it isn't common that I'll have the compose box open and have text in it, unless I'm actively thinking about an in-progress message. But it is common to have the compose box open and empty. So this is a way of resetting the state to "compose box is closed", without first looking to see if it is.
