Topic: Ease of moderation

Yash RE (Mar 28 2020 at 09:19):

I've been thinking about our UX and our messaging model in terms of how easy it is to moderate large groups of users (>1000) actively talking to each other.

The context is that I've been made a mod in a small discord server (and so I'm seeing the interface and experience) and I'm also a member of a different, active server with thousands of users that kinda struggles with moderation. The big community tends to attract somewhat toxic people at times and the moderation strategy is basically "well if you go into the more risky channels, you might not get help immediately" but there are lower traffic channels that are moderated heavily.

However, I understand that our user base involves more trust than "random trolls on the internet" ie we cater more to businesses / companies / open source communities than hobbyist groups and fan-base communities. This is a pretty strong assumption to the Zulip interface and experience... but then I wonder if we'll be questioning it more in the future... for example consider the change of going from an always public email to hidden email setting. I wonder if there's more optional controls that we could add to cater to more people's needs... and if there's more core changes that we could make which would be convenient for users from any kind of userbase. Which brings this topic back to the "moderator user experience".

In general, I think our messaging model would make the experience easier than other places, because it's simpler to go into a topic and say "this entire topic should go over there" than to say "hey guys, this conversation you're having should go into that channel". Moving topics is a v. big improvement from this perspective.

Interesting to think about :thinking:.
