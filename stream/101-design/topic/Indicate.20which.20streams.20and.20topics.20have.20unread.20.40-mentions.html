Tim Abbott (Aug 25 2022 at 23:57):

So we have @ icons appearing in ~4 places in the app:

  • What you type in the compose box, by definition an @ character.
  • Actual mention pills, currently an @ character.
  • /#settings/notifications, currently an @ character.
  • The left sidebar "Mentions" view icon, currently fa-at
  • User popovers "Copy mention syntax" element, currently fa-at

This feature prototype uses an @ character, which I think does look significant better than fa-at in this context.

I propose we not block this feature on aligning those icons with each other, and instead do a follow-up PR that changes all the @s we have in the app to the same thing. Doing so would save me significant integration headaches.

I think the @ character is fairly defensible since it looks decent and will match what you type in the compose box, or we can use the Feather icon; I don't like the Font Awesome one, so would not push for that. But I think it's reasonable to think of the project to create sanity in our @s across the app as its own mini project.
