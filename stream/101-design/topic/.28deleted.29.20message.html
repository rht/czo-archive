Tim Abbott (May 11 2021 at 15:47):

We might also want to tag them for translation. But I guess I'm not sure this sort of local adjustment is the right approach. Structurally, the "message content is '(deleted)'" case is one that exists when our permissions configuration is to not allow users to fully delete messages.

This model is designed to preserve the edit history, so that one can investigate reported bad behavior that was since deleted (etc.). It's possible that we should instead be thinking about making that configuration more rare and/or allowing users to delete their own messages in chat.zulip.org.
