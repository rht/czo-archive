Steve Howell (Mar 23 2020 at 14:03):

It's worth listing all the commands:

  • /me - clearly messaging-related
  • /poll, /todo - makes the most sense in the compose box, because you're addressing the poll to a particular stream or PM group, plus you can type the title and options right there
  • /tictactoe - for this and other future toys we build, the compose box makes sense since the game gets embedded in a message
  • /day, /night, /light, /dark - really have nothing to do with the compose box (and can be set in settings)
  • /ping - not super important, it's just a way to test slash commands, but I'd keep it in the compose box

Given that only really day/night make sense out of the compose box, and we already have a way to do the relevant actions via the settings menu, I'm not sure how much value another UI would add here.

Steve Howell (Mar 23 2020 at 14:05):

I actually do support building some kind of command pallet, but I think that's mostly orthogonal to the slash commands. If we have a bunch of useful commands that we want to have handy at the press of a key, and for which there's not an already logical place to put them (e.g. sidebar menus, message menus, etc.), then we should make a little pallet menu. But we should first enumerate what those tasks are.

Steve Howell (Mar 23 2020 at 14:06):

(We may soon have a "/full" slash command as well, to add to the list above. It's in the same category as the day/night stuff.)

Steve Howell (Mar 23 2020 at 14:10):

For me a command menu would mostly include things you can do with hotkeys. Instead of having to memorize q, w, n, and p, I'd want a menu like this that I could navigate with error keys:

  • toggle to night mode
  • go to next unread topic (n)
  • go to next unread PMs (p)
  • search streams (q)
  • search users (w)
  • start poll (/poll)
  • view messages sent by me

Steve Howell (Mar 23 2020 at 14:11):

And the idea here is that the command pallet would also reinforce the hotkeys and other UI idioms, so as I get more advanced, I'll just start hitting "n" directly or just type "/poll" directly into the compose box.
