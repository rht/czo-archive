Tim Abbott said:

I think that's probably a useful step, though I'll note that we do not intend for user groups to have long descriptions, so the screenshot is kinda misleading. In general I want to do a more holistic redesign of the user groups setting UI to perhaps be more similar to the two-panel "Stream settings" UI, so I don't want to necessarily go too deep into tweaking the current system.

I guess in particular:

  • We want to be able to browse user groups using filters etc. like our other table-based components.
  • We want the set of members for user groups to be browsable like stream subscribers.
  • We want to be able to support multiple screens worth of user group settings, such as a list of permissions the group has.
  • We want to be able to browse subgroups.

Alya Abbott (Mar 18 2022 at 18:58):

The broader context is that in #19525 we plan to make user groups much more powerful, and we'll need a UI for viewing and managing the new functionality.

Alya Abbott (Mar 18 2022 at 19:02):

A proposal is to make the groups management UI similar to the stream management UI, including:

  • List of groups on the left, with two tabs (all groups, and groups to which the user belongs).
  • Right-side panel for viewing and managing a specific group.
  • Tabs within the right-side panel for the things Tim Abbott mentioned above, i.e. something like "General", "Personal", "Permissions", and "Subscribers".

I'll spend more time to hammer out the specifics after 5.0, but do folks think the general direction makes sense?
