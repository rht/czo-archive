Daniil Fadeev (Jul 12 2023 at 08:13):

I like the idea of having buttons appear on hover. It helps keep things clear when you don't want to take any actions with the topic. We should also think about adding a delay on hover to avoid showing the buttons when you're just moving the mouse around.

We need to decide how many buttons to show. Personally, I'd prefer if all the possible buttons are available on hover for fast action. But if we want to collapse them, I see three options:

  1. A three-dot popover menu.
  2. An arrow to expand/collapse the buttons.
  3. Horizontal scroll.

All three options are good, but I prefer the collapse/expand button option. It looks visually stylish, can be smoothly animated, and we can store the state so users can expand it if they find it helpful and keep it expanded for the session, at least.

Daniil Fadeev (Jul 12 2023 at 08:20):

I'm not sure if we should display the muted status using text when we already have all the controls in icons. I find inspiration from the macOS menu bar, where you have all the controls represented by icons and other widgets. The muted or sound state is simply indicated by different icons, and I find it really helpful to have control and easily know the state of things in one place.

Aman (amanagr) (Jul 12 2023 at 10:32):
