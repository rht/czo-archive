Abhijeet Bodas (Jun 28 2023 at 06:30):

I wanted to discuss the default values for when to automatically follow topics. The issue which Alya opened (#25914) mentions the following:

Option 1:
Label: Automatically follow topics
Options: Topics I participate in, Topics I start [default], Never

IMO, the default for this new setting should be the NEVER value, for two reasons:

  1. If, after a server upgrade, user start getting notifications all of a sudden (when they didn't change any notification settings), that could get annoying. And these are going to be a lot of notifications in number (much more than mentions/PMs, for which we have the default notification setting as ON).
    Setting this to NEVER might hinder feature discovery, but I think we can find other ways to advertise this feature (for example, in settings/topics, or in the topic popover)

  2. The other (technical) reason is that having the default as "Topics I start" skews a lot of our tests, which now need to take into account extra events and notifications. To avoid this, I suggested we set the value to NEVER for our test users, but even that is a bit hacky, since it means the real scenario is not reflected in the tests.
