Steve Howell (Sep 08 2017 at 18:02):

I can answer #2 first. We don't use Zulip instead of GitHub; we use it in addition to Github. For rapid-fire discussion, where maybe you're just polling people for ideas on how to fix something, people prefer the Zulip UI to GH. Once we reach consensus, we usually cross-post something to GH.

Steve Howell (Sep 08 2017 at 18:03):

For #1, I am wondering how we can make the sidebar UI a little more obvious in showing topics.

Cynthia Lin (Sep 08 2017 at 18:04):

re: #2: In addition to what Steve said, GitHub comments produce a lot of unnecessary emails that bother some contributors, so we prefer long, real-time discussion here :)

Rishi Gupta (Sep 08 2017 at 19:28):

Thanks for the feedback Ariel! Definitely post as you have more thoughts.

For #1: Hopefully this will feel more natural as you have more unread messages? I agree we should make the onboarding of this more intuitive, and there are some open PRs in flight that will hopefully do so.

For #2: Discussion on zulip/zulip issues averages 100s of messages a day (now is currently the lowest its been in months, since many of the core maintainers are on vacation). It would be hard for most contributors to manage 100s of emails or 100s of github comments, but it's easy to manage 100s of messages a day in Zulip.

For "what goes here vs there", by default it should always go on chat.zulip.org. We should clarify that in our contributors guide.
