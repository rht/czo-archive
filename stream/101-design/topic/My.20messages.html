Edmund Lian (Jun 16 2017 at 00:14):

When I want to see all the (public) messages that another user has send, I use a pull-down next to the other user's name, and then select "View messages sent".

If I want to see all the messages I have sent, I either have to:

1. Enter the search string "sender:me" in the appropriate stream (or home).
2. Click on my own name in any message I've sent, then select "View messages sent".

Both methods are not all intuitive. It took quite a while to figure this out. It would be much more obvious and consistent to include either my own name, or "me" in the list of users so that the same approach is used across all users, to find private messages, sent messages, etc.

Steve Howell (Jun 16 2017 at 00:28):

The original thought process behind removing the user from their own buddy list was this:

  • It was a slight waste of space, since you already knew you were online.
  • We were planning to have a special widget for the user to do "me" kind of things (Rishi gave a good example, which is setting your away status, plus maybe show your avatar).

We haven't gotten around to having the special widget. I don't think the space savings is that big a concern. So it's possible we should do either of the following:

  • Just add the user back to the buddy list (it's about 15 lines of easy code to change, including tests, I'm guessing).
  • Build a nice widget for the user, showing their avatar, having a dropdown to view messages they sent, and then have room to build out future bells and whistles like setting away statuses, etc.)
