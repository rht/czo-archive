Tim Abbott (Aug 05 2022 at 23:22):

Specifically, when we change options for a select field, the a saving..saved loading spinner experience happens in the modal, but it's too quick to see. This contrasts with, for example, deleting a custom profile field or reordering them, which shows our standard saving...saved organization settings experience. I propose the following changes from testing this:

  • We add a success_continuation when closing these modals after a successful change to show the "Saved!" notice on the main settings page, for consistency.
  • We change the "delete a custom profile field" button to have a confirmation modal showing how many users have set that field, just like we added recently for removing an option for the select field type, rather than just immediately executing the deletion.
  • I noticed the modal to confirm deleting an option from a select type custom profile field doesn't tell you which option(s) you removed; maybe it could be extended to address that?
