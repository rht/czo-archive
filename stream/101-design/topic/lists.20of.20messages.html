Stream: design

Topic: lists of messages

Steve Howell (Jan 04 2018 at 20:45):

Something that's come up lately is that we have several places in our UI where we have (or will soon have) a list of messages that don't truly correspond to a "message feed" UI, i.e. you're not really viewing messages in the context of their original conversations:

  • Starred messages
  • Mentions
  • keyword search results
  • drafts
  • scheduled messages (in a PR now)

For the first three, we show is:starred, is:mentioned, and search:lunch as typical narrows, which can create confusion when you forget you're in that unique kind of narrow.

For drafts, we still have our V1 UI that tries to render drafts as messages, but I find it very visually confusing.

And for scheduled messages, we don't even have a proposed UI yet, but it made me think of the other stuff.

Steve Howell (Jan 04 2018 at 20:47):

So I'm wondering if we want some kind of unified way of dealing with "lists of messages" that aren't single conversations or even interleaved conversations.

It sort of depends on what you mean by "cards," but I think a key thing would be to have visual separation (i.e. whitespace) between the messages, since adjacent messages in things like "Starred messages" aren't any more closely related to each other than non-adjacent messages.
